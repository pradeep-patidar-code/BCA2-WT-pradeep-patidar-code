## What Each Folder Contains

### 📂 `2-Content/0-Instructions-and-Prerequisites/` ← You are here!
The setup guides and help documentation for the course. Contains step-by-step instructions for students to:
- Create GitHub accounts
- Install required software
- Clone the course repository
- Understand the course structure
- Accept and submit lab assignments
- Troubleshoot common problems

Also includes an instructor guide for setting up and managing GitHub Classroom.

### 📂 `1-Curriculum/`
The **official syllabus** documents from the university. Use these to know exactly what topics are covered in your exam:
- `25BCA060T_Web_Technology.md` — Theory topics for exams
- `25BCA060P_Web_Technology_Lab.md` — All 30 lab experiments with requirements
- `8-Self-Study/` subfolder — Foundational prerequisite topics to review before starting

### 📂 `2-Content/`
Main course content organized by teaching unit. Each subfolder contains lecture files and associated lab experiments:

| Folder | Unit Topic | Lecture Files | Lab Experiments |
|--------|-----------|---------------|----------------|
| `0-Instructions-and-Prerequisites/` | Setup guides, GitHub, assignments | — | Help & guides |
| `1-Internet-Technology/` | Internet basics, TCP/IP, HTTP | 01–05 | Theory only |
| `2-HTML-Fundamentals/` | HTML tags, links, images, tables | 06–13 | Labs 1–9 |
| `3-HTML5/` | Semantic elements, forms, canvas | 14–18 | Labs 9, 12, 13 |
| `4-CSS/` | Styling, box model, flexbox, grid | 19–25 | Labs 10, 11 |
| `5-JavaScript/` | Variables, DOM, events, validation | 26–30 | Labs 17–21 |
| `6-Bootstrap/` | Grid, components, responsive | 31–37 | Labs 14–16, 25–30 |
| `7-Web-Forms-and-Database/` | Forms, SQL, authentication | 38–41 | Labs 22–24 |
| `8-Self-Study/` | **Deprecated** — moved to `1-Curriculum/8-Self-Study/` | — | — |

### 📂 `3-Projects/`
Two larger projects that combine everything you've learned:
- **Department Website** — Build a multi-page website for your BCA department
- **E-Book** — Create an interactive e-book/magazine

## How Each Lecture File Is Structured

Every file in the `2-Content/` folder follows the same pattern:

```
# Lecture Title
├── Learning Objectives         ← What you'll learn
├── Theory Section              ← Explanations with analogies and diagrams
│   ├── Sub-topic 1
│   ├── Sub-topic 2
│   └── Code examples
├── Practical Session           ← Lab experiment with full code
│   ├── Instructions
│   └── Complete working code
├── Summary Table               ← Quick reference
└── Self-Assessment Questions   ← Test your understanding
```

## How to Navigate the Course

### If You're Following the Schedule
1. Check [Lecture-Plan.md](../../../Lecture-Plan.md) for today's topic
2. Open the corresponding file in the topic folder (e.g., `1-Internet-Technology/`, `2-HTML-Fundamentals/`)
3. Read the theory, then try the practical exercise

### If You Want to Jump to a Topic
1. Look at the folder names in `2-Content/` — they are numbered by unit
2. Each file inside is numbered sequentially (01, 02, 03...)
3. Open the file for the topic you want

### If You're Preparing for Exams
1. Check [1-Curriculum/25BCA060T_Web_Technology.md](../../../1-Curriculum/25BCA060T_Web_Technology.md) for exact exam syllabus
2. Read the **Summary Tables** at the end of each lecture file
3. Answer the **Self-Assessment Questions** in each file
